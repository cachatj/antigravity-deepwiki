# ANTIGRAVITY AGENT INSTRUCTIONS 

## Project Overview
This repo orchestrates pharma Share of Wallet (SOW) data pipelines using Airflow and Google BigQuery. The architecture is modular: each DAG file (e.g., 00_orchestration_SOW_Master_Monthly.py) represents a distinct pipeline stage (ingestion, processing, pricing, reporting). SQL logic is externalized in the `sql/` directory and injected via Jinja templates.

## Architecture & Data Flow
- **DAGs:** Each DAG is a Python file prefixed by stage (e.g., `01_ingestion_`, `07_04_processing_`). Orchestration DAGs (00, 07_00) trigger downstream DAGs for batch processing.
- **SQL Templates:** All BigQuery queries are stored in `sql/` and referenced via `{% include 'filename.sql' %}`. Never embed raw SQL in Python; always use external files.
- **Environment Variables:** Use Airflow Variables (`ABRACADATA_SOW`) to set `project_id`, `sow_sql_directory`, and other runtime config. Service accounts are selected by `airflow_env` (prd, stg, dev).
- **Service Accounts:** Switch by environment:
  - prd: `sa-phm-pd-psow@edc-p-sow-pr-cah.iam.gserviceaccount.com`
  - stg: `sa-phm-pd-psow@edc-p-sowstg-pr-cah.iam.gserviceaccount.com`
  - dev: `sa-phm-pd-psow@edc-p-sow-np-cah.iam.gserviceaccount.com`
- **Alerting:** All DAGs use `alert_email` for failure notifications.

## Developer Workflow
- **Build/Test:** No explicit build/test scripts; deploy DAGs by copying Python files to Airflow. Use Airflow UI or CLI to trigger DAG runs.
- **Debugging:** Use Airflow logs for task-level debugging. Parameters are passed via `dag_run.conf` or Airflow Variables.
- **SQL Edits:** Edit SQL in `sql/` directory. Reference via Jinja `{% include %}` in DAGs.
- **Adding DAGs:** Copy an existing DAG file as template. Update `dag_name`, `dag_desc`, tags, and SQL includes.
- **Dependencies:** All DAGs depend on Airflow, Google Cloud, and BigQuery Python providers. No custom Python packages outside standard Airflow.

## Project-Specific Conventions
- **Naming:** Prefix files by stage and frequency (e.g., `03_ingestion_FinalDispensing_Weekly.py`).
- **Task Arguments:** Use `default_args` for retries, SLAs, and alerting. Example:
  ```python
  default_args = {
      "owner": "pdna",
      "depends_on_past": False,
      "email": alert_email,
      "email_on_failure": True,
      "email_on_retry": False,
      "retries": 3,
      "retry_delay": timedelta(minutes=15),
      "sla": timedelta(minutes=120),
      "trigger_rule": "all_success",
  }
  ```
- **BigQueryOperator:** Always use SQL file references, not inline SQL.
- **Tags:** Use descriptive tags for DAGs (e.g., `['AbracaDATA', 'Share Of Wallet', 'SOW']`).

## Integration Points
- **Airflow Variables:** Central config via `ABRACADATA_SOW` (JSON). Example keys: `edna_project`, `sow_repository_directory`.
- **BigQuery:** All data processing is via BigQuery. Dataset: `D2_PHM_PD_PSOW`.
- **Email Alerts:** Failure notifications sent to project team.

## Examples
- See [00_orchestration_SOW_Master_Monthly.py](00_orchestration_SOW_Master_Monthly.py) and [template_dag_psow_test.py](template_dag_psow_test.py) for canonical DAG structure.
